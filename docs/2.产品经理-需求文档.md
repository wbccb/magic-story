# 📄 奇思妙画 (MagicStory) V1.0 产品需求文件 (PRD)

- **文件版本**：V 1.0.0
- **撰写日期**：2026年3月14日
- **目标平台**：iOS / iPadOS (优先适配 iPad 大萤幕，兼容 iPhone)
- **产品定位**：结合多模态大模型与专有渲染管线的儿童 AIGC 绘本动画 App。

---

## 一、 产品总览 (Product Overview)

### 1.1 核心价值主张
让 3-8 岁儿童透过「自由涂鸦 + 童音讲述」，在 60 秒内获得一部「角色连贯、运镜专业、配音生动」的专属 3D 微电影，并支援一键印制为 AR 实体绘本。

### 1.2 目标使用者与场景 (User Personas)
*   **终端使用者（儿童，3-8 岁）**：看不懂复杂文字，耐心极低，喜欢即时回馈与视觉刺激。操作以「点击、涂抹、按住说话」为主。
*   **付费决策者（家长，28-40 岁）**：注重孩童隐私，期望产品具有教育（STEAM）意义，愿意为高品质的数位内容及实体纪念品付费。

---

## 二、 核心业务流程 (Core User Journey)

**「魔法创作流 (Magic Flow)」**：
1.  **首页**：点击【巨大魔法棒（新建故事）】。
2.  **分镜创作 (Scene 1~3)**：
    *   在画布上画图（支援防手掌误触）。
    *   按住麦克风说一段故事。
    *   AI 即时将语音转为优美的童话旁白文字（家长可协助修改）。
3.  **魔法生成 (The Rendering)**：点击【变成动画片】。前端进入沉浸式等待动画，后端触发「魔法渲染引擎（四大护城河管线）」。
4.  **放映室 (Playback)**：播放最终生成的 MP4 微电影。
5.  **家长决策 (Parent Zone)**：解锁家长模式（算术题），进行分享、储存至相机胶卷、付费订阅或下单实体 AR 绘本。

## 三、 功能模块详细规格 (Feature Specifications)

### 3.1 儿童创作模块 (Creation Studio)
**目标**：提供极致流畅、无干扰的多模态输入体验。

| 功能名称 | 产品逻辑与规则 | 验收标准 (Acceptance Criteria) |
| :--- | :--- | :--- |
| **超级画布 (Canvas)** | 1. 隐藏所有复杂图层概念。<br>2. 工具列仅保留：5 种基础色、3 种笔刷粗细、橡皮擦、上一步/清空。<br>3. **防误触机制**：若系统侦测到大面积触控（如手掌边缘），自动忽略该输入点。 | 儿童手腕靠在 iPad 萤幕上画画时，不会产生乱线或触发画面缩放。笔触必须有平滑的压感变化。 |
| **童音录制 (Voice Input)** | 1. 底部超大麦克风按钮，支援长按录音或点击切换录音状态。<br>2. 录音时周围有明显的音波呼吸灯效。<br>3. 限制单页录音长度最长 15 秒。 | 收音清晰，且在背景有轻微电视声音的环境下，仍能透过前端降噪演算法提取人声。 |
| **多模态语义纠错** | 1. 录音结束后，背景非同步发送 `Audio + Canvas Image` 给后端 VLM。<br>2. 画面弹出「✨ 魔法精灵正在听...」，2 秒后显示修正后的优美旁白。<br>3. 提供一个【编辑】小按钮供家长修改文字。 | 故意说含糊的字眼（如「粗苹果」），若画布上有苹果形状，系统必须正确识别并输出为「吃苹果」。 |
| **多页管理** | 1. 画面边缘提供【下一页】按钮。<br>2. V1.0 版本限制一个故事最多 3 页（控制伺服器成本）。 | |

### 3.2 魔法渲染引擎 (Magic Rendering Engine - 后端 AI 逻辑映射)
**目标**：将 BP 中的「技术护城河」转化为稳定的产品体验，确保生成成功率 > 95%。

| 核心管线 (AI Moats) | 触发条件与产品逻辑 | 边缘案例处理 (Fallback) |
| :--- | :--- | :--- |
| **1. 智能渲染路由** | **AI 产品策略**：由后端 LLM 解析旁白。若判断无复杂动作（如「这是一座山」），则走 2.5D 视差渲染；若有动作（如「恐龙在跑」），才呼叫 I2V 大模型。 | 若 I2V 呼叫逾时（超过 40 秒），立刻降级触发 2.5D 渲染管线，确保用户在 60 秒内一定能拿到影片。 |
| **2. 角色一致性锁定** | **AI 产品策略**：第一页生成的 3D 主体，后端自动提取为 Reference Image。第 2、3 页的图像生成必须强制挂载该 Image Prompt。 | 若 VLM 判断第 2 页角色与第 1 页相似度过低，自动重绘第 2 页（至多 1 次）。 |
| **3. AI 导演中枢** | 后端隐藏执行，将使用者的旁白扩写为带有 `Camera Tracking`、`Lighting` 等参数的专业 Prompt。使用者无感知。 | 无。 |
| **4. VLM 视觉巡检兜底** | **最核心的 UX 保障**：I2V 生成后，后端 VLM 检查是否出现肢体扭曲或破图。 | **[Silent Retry 机制]**：若破图，使用者端动画显示「魔法正在加强...」，后端重新抽卡 1 次。若再失败，降级为静态 3D 图 + 镜头平移特效。**绝对不能让儿童看到崩坏的恐怖画面。** |

### 3.3 等待与播放模块 (Playback & AR)
**目标**：化解长达 30-60 秒等待期的焦虑，提供惊豔的观影体验。

| 功能名称 | 产品逻辑与规则 | 验收标准 (Acceptance Criteria) |
| :--- | :--- | :--- |
| **魔法沉浸等待页** | 1. 不能使用无聊的进度条。改为全萤幕的动画（如一只吉祥物在熬制魔法药水）。<br>2. 动态文字提示：「正在为恐龙上色...」、「正在请配音员准备...」。 | 确保等待的 60 秒内，孩童的注意力能被动画吸引而不关闭 App。 |
| **专属放映室** | 1. 横式全萤幕播放 MP4 影片。<br>2. 影片包含：AI 旁白配音、轻快 BGM、动态画面。<br>3. 右下角常驻 App 的品牌浮水印（非付费会员）。 | 影片音画同步，BGM 音量不可盖过旁白。 |

### 3.4 家长控制区与商业化 (Parent Zone & Monetization)
**目标**：遵守 COPPA 隐私法规，并实现 SaaS 订阅与实体电商转化。

| 功能名称 | 产品逻辑与规则 | 验收标准 (Acceptance Criteria) |
| :--- | :--- | :--- |
| **家长门 (Parental Gate)** | 进入【家长中心】、【分享】或【储存】前，必须弹出乘法算术题（如 `8 x 7 = ?`），答对才能进入。 | 确保 3-8 岁儿童无法自行点击付费或将资料分享至外部社群。 |
| **SaaS 订阅 (Magic Pass)** | 1. **免费版**：每月限生成 5 个故事，仅基础 2.5D 动态，有浮水印。<br>2. **付费版（大陆 ¥29-39/月，台湾 NT$150/月，海外 $4.99/月）**：解锁无限次 3D 皮克斯风 I2V 生成、移除浮水印、解锁「家长声音复制 (Voice Clone)」功能。 | 串接 Apple IAP (In-App Purchase) 订阅机制，支持多区域定价。 |
| **实体绘本印制 (O2O)** | 1. 在作品集列表中，提供【印成实体绘本】按钮。<br>2. 串接第三方印刷厂 API 或提供表单填写收件地址。<br>3. 支援绿界/Line Pay 支付单次印刷费 (NT$ 600)。 | 付费流程顺畅，且能预览实体书的 3D 翻页效果。 |

## 四、 隐私、合规与非功能性需求 (NFRs)

### 4.1 儿童隐私合规 (COPPA & GDPR-K 严格遵守)
做为 AI PM，这是我对研发团队的最高要求，违规将导致 App 被苹果下架：
1.  **无感註册**：V1.0 支援 Device ID 匿名使用，不强制作为儿童的实名註册。
2.  **音档与影像销毁机制**：儿童上传的原始音档 (`.wav`) 及照片，在生成完毕并提取出文字/特征后，**伺服器端必须在 24 小时内实体删除 (Hard Delete)**，绝不可用于训练底层开源大模型。
3.  **隐私声明**：首次启动必须由家长（通过算术题）同意《儿童隐私保护条款》。

### 4.2 效能与基础建设需求
1.  **生成延迟 (Latency)**：从点击生成到影片开始播放，P90 延迟必须控制在 **60 秒** 以内。
2.  **App 启动时间**：冷启动 < 2 秒。
3.  **离线支援**：画布与录音功能需支援离线操作（暂存至本地 SQLite），待连上 Wi-Fi 后再点击生成。

## 五、 数据埋点与成功指标 (Metrics & KPIs)

为了验证我们的「魔法渲染管线」是否真的降低了成本并提升了体验，上线后需追蹤以下指标：

### 5.1 核心商业指标 (North Star Metrics)
*   **生成转换率 (Generation Conversion Rate)**：点击「新建故事」到最终成功看到影片的使用者比例（目标 > 80%）。
*   **免费转付费率 (Free-to-Paid CVR)**：免费用户触发「每周 2 次限制」后，订阅 Magic Pass 的比例（目标 > 5%）。
*   **实体绘本附加购买率**：生成满 3 个故事的用户中，下单印制实体绘本的比例。

### 5.2 AI 工程健康指标 (AI Pipeline Health)
*   **智能路由分布率**：追蹤 2.5D、2D 骨骼、3D I2V 的实际触发比例。（理想模型：50% 走 2.5D，30% 走 2D，20% 走昂贵的 I2V，以维持健康的单位经济模型）。
*   **VLM 兜底触发率 (Fallback Rate)**：后端巡检发现破图并自动重试/降级的机率。（若此数值 > 15%，代表底层 I2V 模型的 Prompt 策略需要优化）。
*   **平均单次生成成本 (Unit Cost)**：伺服器 GPU 算力 + API 呼叫费 / 成功生成的影片数。（目标控制在 NT$ 3 元以内）。

## 六、 发布计画 (Release Plan)

*   **Milestone 1 (Week 1-3)**：完成本地端画布、录音模块与家长门 UI。
*   **Milestone 2 (Week 4-6)**：后端「魔法渲染引擎」四大管线 API 串接与极限边缘测试（故意输入各种噪音和乱画）。
*   **Milestone 3 (Week 7)**：TestFlight 内部测试（邀请 20 位 3-8 岁儿童进行易用性测试）。
*   **Milestone 4 (Week 8)**：提交 App Store 儿童类别审核，正式上线。
