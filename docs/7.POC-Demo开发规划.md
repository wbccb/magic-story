# 奇思妙画 POC Demo 开发规划

## 1. POC 目标

用最短路径验证一个核心命题：**儿童涂鸦 + 语音 → AI 动画短片，60 秒内出片，单次成本 < $0.25**。

不验证的事项：用户体系、订阅付费、实体绘本、离线模式、家长门、角色一致性、App Store 上架。

## 2. POC 范围裁剪

| 维度 | MVP 方案 | POC 裁剪 |
| :--- | :--- | :--- |
| 客户端 | React Native / 微信小程序 | 纯 H5 单页应用 (Vue 3 / React) |
| 页数 | 3 页故事 | **1 页**（单图单音 → 单段视频） |
| 画布 | RN Skia + Palm Rejection | HTML Canvas + 基础画笔 |
| 录音 | expo-av + 呼吸灯动画 | Web MediaRecorder API |
| 用户体系 | Device ID + JWT | 无认证，匿名访问 |
| 角色一致性 | SAM 抠图 + IP-Adapter | 跳过（单页无需跨页一致） |
| 渲染路由 | 三路由 (2.5D / 骨骼 / I2V) | **仅两路由**: 2.5D 视差 + I2V |
| VLM 巡检 | GPT-4o-mini 3帧检测 | 跳过（人工目检） |
| TTS | Edge-TTS 童声 | Edge-TTS（保留，成本为零） |
| 音视频合成 | FFmpeg 多轨拼接 | FFmpeg 单轨合成（1段视频+1段音频+BGM） |
| 等待页 | Lottie 沉浸动画 | 简单进度条 + 文字提示 |
| 存储 | PostgreSQL + OSS | SQLite + 本地文件系统（或 OSS） |
| 部署 | 云服务器集群 | 单机 Docker Compose |

## 4. 技术选型

| 组件 | 选型 | 理由 |
| :--- | :--- | :--- |
| 前端 | Vite + Vue 3 + TypeScript | 轻量快速，单文件组件，开发体验好 |
| 画布 | HTML5 Canvas + perfect-freehand | 浏览器原生，无需任何框架依赖 |
| 录音 | MediaRecorder API (WebM/WAV) | 浏览器原生，零依赖 |
| 后端 | Python 3.11 + FastAPI | 与 AI 生态无缝衔接 |
| 任务队列 | Celery + Redis | 异步渲染，进度可查 |
| 数据库 | SQLite | POC 够用，零运维 |
| AI 图像 | ComfyUI (RunPod Serverless) | 与 MVP 一致，验证真实管线 |
| I2V | Kling API (单通道) | POC 阶段单通道足够 |
| TTS | Edge-TTS | 免费，中文童声质量可接受 |
| 视频合成 | FFmpeg | 与 MVP 一致 |
| 部署 | Docker Compose (单机) | 一键启动全部服务 |

## 5. 数据模型 (SQLite)

```sql
-- 极简，仅两张表
CREATE TABLE stories (
    id TEXT PRIMARY KEY,           -- UUID
    status TEXT DEFAULT 'draft',   -- draft | uploaded | rendering | completed | failed
    original_image_path TEXT,      -- 原始涂鸦本地路径
    original_audio_path TEXT,      -- 原始录音本地路径
    transcribed_text TEXT,         -- ASR 转写
    corrected_narration TEXT,      -- LLM 修正旁白
    complexity_score INTEGER,      -- 路由评分
    render_route TEXT,             -- parallax_2d | i2v_api
    redrawn_image_path TEXT,       -- ComfyUI 重塑图
    video_clip_path TEXT,          -- 动态视频片段
    final_video_path TEXT,         -- 最终合成 MP4
    progress_pct INTEGER DEFAULT 0,
    current_step TEXT,
    error_message TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE render_costs (
    id TEXT PRIMARY KEY,
    story_id TEXT REFERENCES stories(id),
    step_name TEXT,                -- asr | llm_route | comfyui | i2v | tts | ffmpeg
    cost_usd REAL,
    duration_ms INTEGER,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 6. API 接口 (4 个)

| 方法 | 路径 | 说明 | 请求/响应 |
| :--- | :--- | :--- | :--- |
| POST | `/api/upload` | 上传涂鸦+录音 | multipart: `image`(PNG) + `audio`(WAV) → `{ story_id }` |
| POST | `/api/render/{story_id}` | 触发渲染 | 无 → `{ task_id }` (202) |
| GET | `/api/status/{story_id}` | 查询进度 | → `{ status, progress_pct, current_step, step_message }` |
| GET | `/api/result/{story_id}` | 获取结果 | → `{ final_video_url, narration, cost_breakdown }` |

## 7. 渲染管线 (简化版)

```
上传图片+音频
    │
    ▼
Step 1: ASR 语音转文字 (Whisper API)                    [10%]
    │   耗时: ~2s, 成本: ~$0.006
    ▼
Step 2: LLM 语义纠错 + 路由评分 + 导演Prompt (GPT-4o-mini) [20%]
    │   输入: (原图, 转写文字)
    │   输出: corrected_narration, complexity_score, director_prompt
    │   耗时: ~2s, 成本: ~$0.003
    ▼
Step 3: ComfyUI 图像重塑 (RunPod GPU)                   [40%]
    │   ControlNet Scribble + Flux → 3D黏土风
    │   耗时: ~8-12s, 成本: ~$0.03
    ▼
Step 4: 动态生成 (Smart Router)                          [70%]
    │   ├─ score ≤ 5 → 2.5D Ken Burns (FFmpeg)
    │   │   MiDaS深度图 + zoompan, 耗时: ~3s, 成本: $0.005
    │   └─ score > 5 → I2V API (Kling)
    │       耗时: ~15-40s, 成本: ~$0.15
    │       超时45s → 降级为2.5D
    ▼
Step 5: TTS 配音 (Edge-TTS)                             [85%]
    │   耗时: ~2s, 成本: $0
    ▼
Step 6: FFmpeg 合成                                      [95%]
    │   视频 + TTS旁白 + BGM → MP4
    │   耗时: ~3s, 成本: $0
    ▼
完成 → 返回 MP4 URL                                     [100%]

预估总耗时: 20-60s
预估总成本: $0.04 (2.5D路由) ~ $0.19 (I2V路由)
```

**验收清单**:

##### 任务 P1.2: H5 画布与录音
**开发内容**:
- HTML5 Canvas 触控/鼠标画板
- 集成 perfect-freehand 平滑笔触
- 工具栏: 5 色 + 3 粗细 + 橡皮擦 + 撤销 + 清空
- MediaRecorder 录音 (限时 15s)
- 画布导出 PNG + 录音导出 WAV/WebM
- 上传至后端

**开发步骤**:
1. 创建 `DrawingCanvas.vue` 组件
   - `pointerdown/move/up` 事件收集触控点
   - `perfect-freehand` 生成平滑 SVG Path → Canvas 绑定渲染
2. 工具栏组件: 颜色选择器 + 粗细滑块 + 功能按钮
3. 创建 `AudioRecorder.vue` 组件
   - `navigator.mediaDevices.getUserMedia({ audio: true })`
   - MediaRecorder 录制，15s 自动停止
   - 录音时显示时长倒计时 + 简单音波动画 (CSS)
4. 导出逻辑: `canvas.toBlob()` → PNG, `MediaRecorder.ondataavailable` → Blob
5. 上传: `FormData` 包含 image + audio → `POST /api/upload`

**验收清单**:
- [ ] PC 鼠标和移动端触控均可流畅画线
- [ ] 笔触有平滑的粗细压感效果
- [ ] 5 色切换、撤销、清空正常
- [ ] 录音 15s 自动停止，有倒计时
- [ ] 上传成功返回 story_id

#### Phase 2 (Day 3-5): 后端 AI 管线

##### 任务 P2.1: ASR + LLM 语义纠错 + 路由
**开发内容**:
- Whisper API 语音转文字
- GPT-4o-mini 多模态纠错 (图+文 → 修正旁白)
- 同一次 LLM 调用同时输出: 修正旁白 + complexity_score + director_prompt

**开发步骤**:
1. `asr.py`: 调用 OpenAI Whisper API，输入音频 → 输出文字
2. `llm.py`: 单次 GPT-4o-mini 调用，Prompt 包含:
   - 系统角色: 儿童动画导演助手
   - 输入: 原图(base64) + ASR文字
   - 输出 JSON: `{ corrected_narration, complexity_score, render_route, director_prompt_en }`
3. 在 `tasks.py` 中定义 Celery 任务 `task_parse_intent`
4. 成本记录写入 `render_costs` 表

**验收清单**:
- [ ] 输入含噪音的儿童语音，ASR 能转出可理解的文字
- [ ] "粗苹果" + 苹果图 → 纠正为"吃苹果"或"红苹果"
- [ ] 静态描述 score ≤ 5，动作场景 score > 5
- [ ] director_prompt 包含 Subject/Action/Camera/Lighting

##### 任务 P2.2: ComfyUI 图像重塑
**开发内容**:
- RunPod Serverless 部署 ComfyUI
- Workflow: ControlNet Scribble + Flux/SDXL → 3D 黏土风
- 封装 HTTP 调用客户端

**开发步骤**:
1. RunPod 创建 Serverless Endpoint，加载模型:
   - Flux.1-dev / SDXL
   - ControlNet Scribble
2. 编写 ComfyUI workflow JSON (API 格式)
3. `comfyui.py`: 提交 workflow → 轮询状态 → 下载输出图
4. 超时 30s → 返回原图 + 风格滤镜作为降级

**验收清单**:
- [ ] 输入儿童涂鸦 → 输出 3D 黏土风格图片
- [ ] 推理耗时 < 15s
- [ ] API 不可用时降级不阻塞

---

##### 任务 P2.3: I2V + 2.5D 视差双路由
**开发内容**:
- Kling I2V API 对接
- 2.5D Ken Burns: MiDaS 深度图 + FFmpeg zoompan
- 路由分发逻辑

**开发步骤**:
1. `i2v.py`: Kling API 调用，提交图片+prompt → 轮询 → 下载视频
   - 45s 硬超时，超时触发降级
2. `ffmpeg.py` 中实现 `render_parallax()`:
   - MiDaS 生成深度图 (可用 `torch.hub` 加载轻量模型)
   - FFmpeg zoompan + displace 生成 5s 视差视频
3. `tasks.py` 中定义 `task_generate_video`:
   - `score > 5` → 调用 I2V
   - `score ≤ 5` → 调用 2.5D 视差
   - I2V 失败/超时 → 自动降级为 2.5D

**验收清单**:
- [ ] I2V 路由: 输入重塑图 + prompt → 输出 5s 动态视频
- [ ] 2.5D 路由: 输入重塑图 → 输出 5s 视差运镜视频，耗时 < 5s
- [ ] I2V 超时 → 自动降级为 2.5D，无需人工干预

#### Phase 3 (Day 6-7): 合成与联调

##### 任务 P3.1: TTS + FFmpeg 最终合成
**开发内容**:
- Edge-TTS 中文童声配音
- FFmpeg 单轨合成: 视频 + TTS + BGM → MP4

**开发步骤**:
1. `tts.py`: `edge-tts` 库，voice=`zh-CN-XiaoxiaoNeural`，输入文字 → 输出 MP3
2. 准备 3 首免版权 BGM (欢快/温馨/冒险)
3. `ffmpeg.py` 中实现 `compose_final()`:
   ```
   视频片段 (5s)
   + TTS 旁白 (按实际时长)
   + BGM (volume=30%, 循环至视频结束)
   → 输出 720p H.264 MP4
   ```
4. 视频时长以 TTS 为准: 若 TTS > 5s 则循环视频，若 TTS < 5s 则减速视频

**验收清单**:
- [ ] TTS 童声自然，语速适中
- [ ] BGM 不盖过旁白
- [ ] 最终视频音画同步，无黑帧
- [ ] 输出 MP4 可在浏览器直接播放

| VLM 巡检 | 人工目检 | GPT-4o-mini 自动巡检 + Silent Retry |
| 2D 骨骼动画 | 无 | Animated Drawings 中间路由 |

| 订阅付费 | 无 | Apple IAP / 微信支付 |
| 客户端 | H5 | 微信小程序 / React Native App |
| 沉浸等待页 | 进度条 | Lottie 魔法动画 |
