### 1.2 技术选型与决策依据

| 层级 | 技术选型 | 决策理由 |
| :--- | :--- | :--- |
| **客户端 (Phase 1)** | 微信小程序 (Taro/uni-app) | 零安装门槛，微信生态裂变，快速验证 MVP |
| **客户端 (Phase 2)** | React Native (Expo) + RN Skia | 跨平台一套代码，Skia 提供原生级画布性能 |
| **API 服务** | Python 3.11 + FastAPI | 异步原生支持，与 AI/ML 生态无缝衔接 |
| **任务队列** | Celery + Redis | 成熟的分布式任务调度，支持任务链/组/回调 |
| **AI 图像管线** | ComfyUI (Serverless GPU) | 可视化节点编排，社区生态丰富，支持热更新 workflow |
| **视频合成** | FFmpeg (subprocess) | 工业级音视频处理，零额外成本 |
| **数据库** | PostgreSQL | JSONB 支持灵活 schema，成熟可靠 |
| **对象存储** | 阿里云 OSS / AWS S3 | 生命周期策略自动清理，CDN 加速分发 |
| **实时通信** | WebSocket (FastAPI native) | 渲染进度实时推送，低延迟 |

### 1.3 核心数据流

#### 1.3.1 创作流 (Creation Flow)
```
儿童画图 → 本地缓存(SQLite/Zustand) → 儿童录音 → 本地缓存
    ↓ (每页完成后，后台静默预处理)
上传图片+音频 → ASR语音转文字 → VLM多模态语义纠错
    ↓
返回修正后旁白文字 → 前端展示（家长可编辑）
```

#### 1.3.2 渲染流 (Rendering Flow)
```
点击"变成动画片" → 提交 RenderTask → Celery 任务入队
    ↓
┌─ 步骤1: 意图解析 (LLM Router)
│   ├─ 每页评分 complexity_score (1-10)
│   └─ 生成专业 Prompt + 运镜指令
├─ 步骤2: 图像重塑 (ComfyUI)
│   ├─ 第1页: SAM抠图 → 提取角色锚点 (Reference Image)
│   ├─ 第2-3页: IP-Adapter注入角色锚点 + ControlNet约束
│   └─ 输出: 3张 3D黏土风高清图片
├─ 步骤3: 动态生成 (Smart Router)
│   ├─ score < 4 → 2.5D Ken Burns 视差 (FFmpeg, $0.005)
│   ├─ score 4-6 → 2D 骨骼动画 (Animated Drawings, $0.005)
│   └─ score > 6 → I2V API (Kling/Minimax, $0.15)
├─ 步骤4: VLM 视觉巡检 (Auto-QA)
│   ├─ 截取首/中/末3帧 → GPT-4o-mini 检测
│   ├─ PASS → 继续
│   └─ FAIL → Silent Retry 1次 → 再失败则降级为2.5D
└─ 步骤5: 音视频封装 (FFmpeg)
    ├─ TTS 生成旁白音频 → 计算时长
    ├─ 匹配 BGM → 音量混合
    └─ 拼接3段视频 + 音频 → 输出最终 MP4 → 上传 OSS
```

#### 1.3.3 播放流 (Playback Flow)
```
WebSocket 推送 "render_complete" → 客户端获取 MP4 URL
    ↓
全屏横屏播放 → 播放完毕 → 引导分享/保存/印制绘本
```

### 1.4 安全与合规架构

#### COPPA / GDPR-K 合规要求
| 要求 | 实现方案 |
| :--- | :--- |
| 无儿童实名注册 | Device ID 匿名使用，家长账号通过 Parental Gate 绑定 |
| 原始数据销毁 | 儿童上传的原始音频(.wav)和画作，生成完毕后 24h 内 Hard Delete |
| 数据最小化 | 仅保留生成后的成品视频和文字旁白，不保留中间产物 |
| 家长同意 | 首次启动必须通过算术题验证家长身份后同意隐私条款 |
| 内容安全 | VLM 巡检同时检测不适宜儿童的内容，拦截后降级处理 |

#### 数据生命周期管理
```
原始音频/画作 ──(24h)──→ Hard Delete (CRON Job)
中间产物(ComfyUI输出) ──(1h)──→ 自动清理
成品视频(MP4) ──(免费用户30天/付费用户永久)──→ OSS 生命周期策略
用户账号数据 ──(注销后30天)──→ 全量删除
```

## 第二部分：详细设计 (Detailed Design)

### 2.1 数据模型设计 (Database Schema)

#### 核心实体关系
```
User (1) ──→ (N) Story (1) ──→ (N) Scene
  │                │
  │                └──→ (1) RenderTask (1) ──→ (N) RenderStep
  │
  └──→ (1) Subscription
  └──→ (N) PrintOrder
```

#### 表结构定义

**users 表**
```sql
CREATE TABLE users (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    device_id       VARCHAR(128) UNIQUE NOT NULL,    -- 匿名设备标识
    parent_email    VARCHAR(255),                     -- 家长绑定邮箱(可选)
    subscription_tier VARCHAR(20) DEFAULT 'free',     -- free | magic_pass
    monthly_gen_count INT DEFAULT 0,                  -- 当月已生成次数
    monthly_gen_reset_at TIMESTAMP,                   -- 月度计数重置时间
    privacy_accepted_at TIMESTAMP,                    -- 隐私条款同意时间
    created_at      TIMESTAMP DEFAULT NOW(),
    updated_at      TIMESTAMP DEFAULT NOW()
);
```

**stories 表**
```sql
CREATE TABLE stories (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID REFERENCES users(id) ON DELETE CASCADE,
    title           VARCHAR(100),                     -- AI 自动生成的故事标题
    status          VARCHAR(20) DEFAULT 'draft',      -- draft | rendering | completed | failed
    scene_count     INT DEFAULT 0,                    -- 当前页数 (max 3)
    final_video_url TEXT,                             -- 最终 MP4 的 OSS URL
    video_duration  FLOAT,                            -- 视频总时长(秒)
    thumbnail_url   TEXT,                             -- 封面缩略图
    is_watermarked  BOOLEAN DEFAULT TRUE,             -- 是否带水印
    created_at      TIMESTAMP DEFAULT NOW(),
    updated_at      TIMESTAMP DEFAULT NOW()
);
```

**scenes 表**
```sql
CREATE TABLE scenes (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    story_id        UUID REFERENCES stories(id) ON DELETE CASCADE,
    page_number     INT NOT NULL,                     -- 1, 2, 3
    original_image_url TEXT,                           -- 原始涂鸦 OSS URL (24h后删除)
    original_audio_url TEXT,                           -- 原始录音 OSS URL (24h后删除)
    transcribed_text TEXT,                             -- ASR 转写文字
    corrected_narration TEXT,                          -- VLM 修正后的旁白
    redrawn_image_url TEXT,                            -- ComfyUI 重塑后的3D图片
    reference_image_url TEXT,                          -- 角色锚点图 (仅第1页)
    video_clip_url  TEXT,                              -- 该页生成的视频片段
    complexity_score INT,                              -- LLM 路由评分 (1-10)
    render_route    VARCHAR(20),                       -- parallax_2d | skeleton_2d | i2v_api
    director_prompt TEXT,                              -- AI 导演生成的专业 Prompt
    created_at      TIMESTAMP DEFAULT NOW()
);
```

**render_tasks 表**
```sql
CREATE TABLE render_tasks (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    story_id        UUID REFERENCES stories(id) ON DELETE CASCADE,
    celery_task_id  VARCHAR(255),                     -- Celery 任务 ID
    status          VARCHAR(20) DEFAULT 'queued',     -- queued | processing | qa_check | compositing | completed | failed
    current_step    VARCHAR(50),                      -- 当前执行步骤名称
    progress_pct    INT DEFAULT 0,                    -- 进度百分比 (0-100)
    total_cost_usd  DECIMAL(6,4) DEFAULT 0,           -- 本次渲染总成本
    retry_count     INT DEFAULT 0,                    -- 重试次数
    error_message   TEXT,                             -- 失败原因
    started_at      TIMESTAMP,
    completed_at    TIMESTAMP,
    created_at      TIMESTAMP DEFAULT NOW()
);
```

**subscriptions 表**
```sql
CREATE TABLE subscriptions (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID REFERENCES users(id) ON DELETE CASCADE,
    platform        VARCHAR(20) NOT NULL,             -- apple_iap | wechat_pay
    product_id      VARCHAR(100),                     -- Apple/微信产品 ID
    status          VARCHAR(20) DEFAULT 'active',     -- active | expired | cancelled
    expires_at      TIMESTAMP,
    receipt_data    TEXT,                              -- 支付凭证(加密存储)
    created_at      TIMESTAMP DEFAULT NOW()
);
```

**print_orders 表**
```sql
CREATE TABLE print_orders (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID REFERENCES users(id),
    story_id        UUID REFERENCES stories(id),
    status          VARCHAR(20) DEFAULT 'pending',    -- pending | paid | printing | shipped | delivered
    shipping_address JSONB,                           -- 收件地址(加密)
    price_cny       DECIMAL(8,2),                     -- 订单金额
    tracking_number VARCHAR(100),                     -- 物流单号
    created_at      TIMESTAMP DEFAULT NOW()
);
```

### 2.3 四大 AI 管线详细设计

#### 2.3.1 管线一：智能渲染路由 (Smart Rendering Router)

**输入**: 每页的 `corrected_narration` + `redrawn_image`
**输出**: `complexity_score (1-10)` + `render_route` + `director_prompt`

**LLM 路由 Prompt 模板**:
```
你是一个儿童动画导演助手。根据以下旁白内容，评估画面的动态复杂度。

旁白: "{corrected_narration}"

请返回 JSON:
{
  "complexity_score": <1-10的整数>,
  "reasoning": "<评分理由>",
  "render_route": "<parallax_2d | skeleton_2d | i2v_api>",
  "director_prompt_en": "<英文专业运镜指令，包含 Subject/Action/Camera/Lighting>"
}

评分标准:
- 1-3分: 纯静态场景描述(如"这是一座山"、"天空很蓝") → parallax_2d
- 4-6分: 简单肢体动作(如"小狗在走路"、"女孩在跳舞") → skeleton_2d
- 7-10分: 复杂物理交互(如"恐龙喷火烧了房子"、"飞机穿过云层") → i2v_api
```

**路由决策流程**:
```
narration + image
       │
       ▼
  LLM 评分 (GPT-4o-mini, ~$0.001)
       │
       ├─ score 1-3 ──→ FFmpeg Ken Burns (2.5D视差)
       │                  - 生成 depth map (MiDaS)
       │                  - zoompan + displace 滤镜
       │                  - 成本: $0.005, 耗时: 3-5s
       │
       ├─ score 4-6 ──→ Animated Drawings (2D骨骼)
       │                  - YOLO 检测人形区域
       │                  - 骨骼绑定 + 预设动作库
       │                  - 成本: $0.005, 耗时: 8-12s
       │
       └─ score 7-10 ─→ I2V API (Kling/Minimax)
                          - 附带 director_prompt_en
                          - timeout: 45s 硬限制
                          - 成本: $0.15, 耗时: 15-40s
                          - 超时降级 → 2.5D视差
```

#### 2.3.2 管线二：角色一致性锁定 (Character Consistency Vault)

**核心流程**:
```
第1页涂鸦
    │
    ▼
ComfyUI Workflow:
    ControlNet Scribble (保持草图骨架)
    + Flux/SDXL (3D claymation style 重塑)
    │
    ▼
SAM (Segment Anything) 自动抠图
    → 提取主体 → 存为 reference_image
    │
    ▼
第2-3页涂鸦
    │
    ▼
ComfyUI Workflow:
    ControlNet Scribble
    + IP-Adapter Plus (注入 reference_image 作为角色锚点, weight=0.6-0.8)
    + Flux/SDXL (同一 seed 基础 + 3D claymation style)
    │
    ▼
VLM 相似度校验:
    截取第N页主体 vs reference_image
    → 相似度 < 阈值 → 自动重绘1次(提高 IP-Adapter weight 至 0.9)
    → 再次失败 → 接受当前结果(宁可略有差异也不无限重试)
```

**ComfyUI Workflow 节点编排**:
```
[Load Image] → [ControlNet Preprocessor (Scribble)] → [Apply ControlNet]
                                                            ↓
[Load Reference Image] → [IP-Adapter Plus] ──────────→ [KSampler]
                                                            ↓
[Positive Prompt: "3D claymation Pixar style, {subject}"] → [VAE Decode]
                                                            ↓
                                                      [Save Image]
```

**关键参数**:
| 参数 | 第1页 | 第2-3页 |
| :--- | :--- | :--- |
| ControlNet strength | 0.7 | 0.5 |
| IP-Adapter weight | N/A | 0.6-0.8 |
| CFG Scale | 7.0 | 6.5 |
| Steps | 25 | 20 |
| Style Prompt | `3D claymation, Pixar style, soft lighting` | 同上 |

#### 2.3.3 管线三：AI 导演编译中枢 (Agentic Director)

**职责**: 将儿童的简单旁白转译为工业级视频生成指令

**转译示例**:
| 儿童旁白 | AI 导演输出 (director_prompt_en) |
| :--- | :--- |
| "恐龙跑了" | `Subject: 3D claymation dinosaur, green body. Action: Running from left to right with bouncy motion. Camera: Panning tracking shot, slight low angle. Lighting: Warm sunset, soft shadows. Style: Pixar animation, motion blur.` |
| "小猫在睡觉" | `Subject: 3D claymation cat, curled up. Action: Gentle breathing motion, tail slight twitch. Camera: Slow zoom in, shallow depth of field. Lighting: Warm indoor, soft golden hour. Style: Cozy storybook illustration.` |
| "飞机飞上天" | `Subject: 3D claymation airplane, colorful. Action: Taking off diagonally upward with trail. Camera: Low angle tracking upward, dramatic perspective. Lighting: Bright blue sky, volumetric clouds. Style: Pixar animation, dynamic motion blur.` |

**Prompt 工程约束**:
- 强制包含 `3D claymation` 风格锁定（提高角色一致性）
- 强制包含 Camera 运镜指令（避免静态画面）
- 禁止出现恐怖、暴力、写实人脸等关键词
- 输出语言固定为英文（I2V 模型对英文 Prompt 理解更好）

#### 2.3.4 管线四：VLM 视觉巡检兜底 (Auto-QA Gatekeeper)

**巡检流程**:
```
I2V 生成的视频片段
       │
       ▼
  FFmpeg 截帧 (第1帧, 中间帧, 最后1帧)
       │
       ▼
  VLM 检测 (GPT-4o-mini, 3张图同时送入)
       │
  Prompt: "检查这3帧动画截图，回答以下问题:
           1. 主体是否出现多余肢体(如3条腿、6个手指)?
           2. 背景是否严重形变或出现不合理元素?
           3. 画面是否包含不适合儿童观看的内容?
           返回 JSON: { pass: bool, issues: string[] }"
       │
       ├─ pass: true ──→ 进入音视频封装步骤
       │
       └─ pass: false ──→ Silent Retry 机制
                              │
                              ├─ retry_count < 1
                              │   → 微调 Prompt (增加负面提示词)
                              │   → 重新调用 I2V API
                              │   → 再次巡检
                              │
                              └─ retry_count >= 1
                                  → 强制降级为 2.5D Ken Burns
                                  → 前端显示"魔法正在加强..."
                                  → 保证一定有输出
```

**巡检成本**: ~$0.005/次 (GPT-4o-mini 3张图片输入)

### 2.4 异步任务编排 (Celery Workflow)

**任务链设计**: 使用 Celery `chain` + `chord` 实现多步骤编排

```python
# 伪代码: 完整渲染任务链
def trigger_render(story_id):
    workflow = chain(
        # Step 1: 并行处理3页的意图解析
        chord(
            [parse_intent.s(scene_id) for scene_id in scene_ids],
            # Step 2: 并行处理3页的图像重塑(依赖Step1的路由结果)
            process_images.s(story_id)
        ),
        # Step 3: 并行生成3页的动态视频
        chord(
            [generate_video.s(scene_id) for scene_id in scene_ids],
            # Step 4: 并行巡检(仅I2V路由的页面需要)
            qa_check_all.s(story_id)
        ),
        # Step 5: TTS 配音生成
        generate_tts.s(story_id),
        # Step 6: FFmpeg 最终合成
        compose_final_video.s(story_id),
        # Step 7: 上传 OSS + 通知客户端
        upload_and_notify.s(story_id)
    )
    workflow.apply_async()
```

**超时与降级策略**:
| 步骤 | 超时时间 | 降级方案 |
| :--- | :--- | :--- |
| 意图解析 (LLM) | 10s | 默认 score=3，走 2.5D |
| 图像重塑 (ComfyUI) | 30s | 使用原始涂鸦 + 风格滤镜 |
| I2V 视频生成 | 45s | 降级为 2.5D Ken Burns |
| VLM 巡检 | 10s | 跳过巡检，直接使用 |
| TTS 配音 | 15s | 使用预置默认童声 |
| FFmpeg 合成 | 20s | 简化转场效果 |

**预处理优化 ("Illusion of Speed")**:
- 儿童画完第1页并录音后，后端立即开始该页的意图解析 + 图像重塑
- 不等用户点击"变成动画片"，偷跑 15-20s 的处理时间
- 实现方式: 前端在 `scene_submitted` 事件后静默调用 `/scenes/{id}/preprocess`

### 2.5 FFmpeg 影音合成管线

**最终合成流程**:
```
输入:
  - 3段视频片段 (scene_1.mp4, scene_2.mp4, scene_3.mp4)
  - 3段TTS旁白 (narration_1.mp3, narration_2.mp3, narration_3.mp3)
  - 1段BGM (bgm_cheerful.mp3)

处理步骤:
  1. 计算每段TTS时长 → 确定每段视频的目标时长
  2. 视频时长调整 (加速/减速/循环以匹配旁白)
  3. 添加转场效果 (crossfade, 0.5s)
  4. 混合音频: TTS音量100% + BGM音量30%
  5. 添加水印 (非付费用户)
  6. 输出: 1080p H.264 MP4, AAC音频
```

**FFmpeg 核心命令模板**:
```bash
ffmpeg -i scene_1.mp4 -i scene_2.mp4 -i scene_3.mp4 \
       -i narration_1.mp3 -i narration_2.mp3 -i narration_3.mp3 \
       -i bgm.mp3 \
       -filter_complex "
         [0:v][1:v]xfade=transition=fade:duration=0.5:offset={t1}[v01];
         [v01][2:v]xfade=transition=fade:duration=0.5:offset={t2}[vout];
         [3:a][4:a][5:a]concat=n=3:v=0:a=1[narration];
         [6:a]volume=0.3[bgm];
         [narration][bgm]amix=inputs=2:duration=first[aout]
       " \
       -map "[vout]" -map "[aout]" \
       -c:v libx264 -preset fast -crf 23 \
       -c:a aac -b:a 128k \
       output.mp4
```

### 2.6 降级与容错策略总览

```
                    正常流程

          ┌────────────┼────────────┐

                              │           │
```

**核心原则**: 无论任何情况，用户在 60 秒内一定能看到一部完整的动画片。最差情况下是 2.5D 视差效果，但绝不会是空白或错误页面。

### 2.7 离线模式与数据同步

**离线能力范围**:
- 画布绘图: 完全离线支持
- 语音录制: 完全离线支持
- 本地缓存: SQLite 存储每页的 `{ image_base64, audio_blob, page_number }`
- AI 生成: 必须联网

**同步策略**:
```
离线创作 → 本地 SQLite 暂存
    │
    ├─ 检测到 Wi-Fi 连接
    │   → 自动上传暂存数据
    │   → 显示"魔法材料已准备好，点击生成动画片"
    │
    └─ 用户点击"生成"时无网络
        → Toast提示"需要连接Wi-Fi才能施展魔法哦"
        → 保留本地数据，不丢失
```
