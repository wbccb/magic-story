# 🛠️ 奇思妙画 V1.0 开发任务与架构蓝图

## 一、 技术堆叠与架构决策 (Tech Stack & Architecture)

在 AI Coding 时代（Cursor / GitHub Copilot），常规的 UI 和 CRUD 接口将由 AI 快速生成，工程师的核心价值在于**架构设计与管线调优**。

*   **App 客户端**：`React Native (Expo)` + `React Native Skia` (处理极致丝滑的画布与压感) + `Zustand` (状态管理)。一套程式码优先编译 iOS/iPadOS。
*   **后端中枢 (API Gateway & Brain)**：`Python 3.11` + `FastAPI` + `Celery` + `Redis` (处理非同步任务伫列，这是绝对核心)。
*   **AI 渲染引擎 (The Engine)**：
    *   工作流引擎：自建 `ComfyUI` 伺服器 (部署于 RunPod 或 Modal 等 Serverless GPU)，处理 ControlNet 与 IP-Adapter。
    *   外部 API 依赖：`GPT-4o` (VLM 意图解析与巡检)、`Kling API` / `Minimax API` (I2V 影片生成)、`Edge-TTS` / `火山引擎` (童声配音)。
*   **影音合成**：后端 `FFmpeg` (结合 Python `subprocess` 动态生成指令)。

## 二、 开发任务分解 (Work Breakdown Structure, WBS)

总开发周期设定为 **8 周 (40 个工作日)**。分为四大模组并行开发。

### 模组 1：App 客户端与极简 UX (开发周期：Week 1-3)
*   **任务 1.1: 核心画布 (Canvas) 开发 (难度：⭐⭐⭐)**
    *   **内容**：基于 `RN Skia` 实作画板。整合 `perfect-freehand` 演算法处理触控点，转换为 SVG Path。实作防手掌误触（Palm Rejection）启发式演算法。
    *   **产出**：支援 5 色、3 笔触、撤销，且在 iPad 上画线毫无延迟的元件。
*   **任务 1.2: 录音与本地状态机 (难度：⭐⭐)**
    *   **内容**：整合 `expo-av` 实作按住录音、放开结束。实作录音时的音波呼吸动画（Framer Motion / Reanimated）。
    *   **产出**：本地 SQLite/Zustand 缓存每页的 `[Base64 图片, 录音 Blob]`。
*   **任务 1.3: 沉浸式等待页与放映室 (难度：⭐⭐)**
    *   **内容**：实作 WebSocket 监听后端渲染进度，对应播放「魔法熬制中」的不同阶段动画。实作全萤幕影片播放器。
*   **任务 1.4: 商业化与合规门禁 (难度：⭐⭐)**
    *   **内容**：实作家长锁（随机乘法题 UI）、Apple IAP 订阅机制串接、隐私政策同意流程。

### 模组 2：魔法渲染引擎 - 核心 AI 管线 (开发周期：Week 2-5)
*(这是本专案的灵魂，占据 60% 开发资源)*

*   **任务 2.1: 智能路由与多模态解析 (难度：⭐⭐⭐⭐)**
    *   **内容**：撰写 `FastAPI` 接口接收音画。呼叫 ASR 将语音转文字。将原图与文字送入 VLM (GPT-4o)，要求返回 JSON（包含修正后的旁白、动态复杂度评分 1-10、提取的 Prompt）。
    *   **产出**：路由决策器（Router），若分数 < 4 走 2.5D，> 4 走 I2V。
*   **任务 2.2: 角色一致性生成 - ComfyUI 工作流 (难度：⭐⭐⭐⭐⭐)**
    *   **内容**：在云端 GPU 搭建 `ComfyUI`。编写一个包含 `SAM (抠图) -> IP-Adapter (角色特征注入) -> ControlNet Scribble (维持草图骨架) -> Flux/SDXL` 的 Workflow API。
    *   **产出**：一个极度稳定的 API，输入草图 + 角色特征图，输出 3D 皮克斯风高画质图片。
*   **任务 2.3: AI 导演与 I2V 动态生成 (难度：⭐⭐⭐⭐)**
    *   **内容**：呼叫 Kling/Minimax API。重点在于撰写强约束的 LLM Prompt，将旁白转译为英文运镜指令（如 `Pan left, motion blur`），传给影片模型。
*   **任务 2.4: 兜底管线 (2.5D Parallax & 2D) (难度：⭐⭐⭐)**
    *   **内容**：实作免 GPU 的降级方案。使用 Python 脚本生成 Depth Map，配合 FFmpeg `zoompan` 与 `displace` 滤镜生成动态影片。

### 模组 3：非同步任务排程与 Auto-QA (开发周期：Week 5-6)
*   **任务 3.1: Celery/Redis 渲染伫列 (难度：⭐⭐⭐)**
    *   **内容**：避免 API Timeout。App 提交任务后获取 `Task_ID`，透过 WebSocket 或 Polling 查询进度。
*   **任务 3.2: 视觉巡检与 Silent Retry 机制 (难度：⭐⭐⭐⭐)**
    *   **内容**：实作 `Gatekeeper` 函数。I2V 影片生成后，自动截取第 1、中、末三帧送入 VLM。
    *   **逻辑**：`if check_hallucination() == True: trigger_fallback_pipeline()`。确保烂片绝对不会回传给客户端。
*   **任务 3.3: 终极影音封装 (难度：⭐⭐)**
    *   **内容**：动态计算 TTS 语音长度，使用 `FFmpeg` 将影片片段、TTS 旁白、BGM 精准对齐并合併为最终的 MP4 档，上传至 AWS S3 / 阿里云 OSS。

### 模组 4：整合测试、效能调优与上线 (开发周期：Week 7-8)
*   **任务 4.1: 极端边缘测试 (Chaos Testing)**
    *   **内容**：故意输入狗叫声、完全黑色的画布、断网重连等，测试系统容错与降级是否平滑。
*   **任务 4.2: 隐私合规与资料清理机制**
    *   **内容**：编写 CRON Job，每天凌晨硬删除（Hard Delete）资料库与 OSS 中超过 24 小时的原始儿童录音与画作，确保符合 COPPA 审查要求。
*   **任务 4.3: App Store 提审与上架**

## 三、 核心开发难点与架构师对策 (Challenges & Mitigations)

投资人看重的是概念，但我们工程师面对的是残酷的物理与算力限制。以下是我们将面临的 3 大生死劫及解决方案：

### 🚨 难点 1：60 秒出片极限 vs. 影片大模型排队延迟
**问题**：呼叫外部 I2V API（如 Kling）在高峰期可能需要排队 2-3 分钟，这会让儿童完全失去耐心，导致产品体验崩溃。
**架构师对策 (The "Illusion of Speed" Strategy)**：
1.  **极致的非同步预处理**：当儿童画完「第一页」并录音结束时，**不要等他们点击「生成影片」，后端立刻在背景偷偷开始生成第一页的 3D 图片和音档**。这能偷出至少 15 秒的时间。
2.  **强制降级**：在 Celery 任务中设置严格的 `timeout=45s`。只要 I2V API 超过 45 秒未回传，立刻斩断进程，无缝切换至 2.5D FFmpeg 视差渲染，确保 60 秒内绝对有影片可以播。

### 🚨 难点 2：角色一致性 (IP Consistency) 在多页故事中的崩坏
**问题**：目前没有任何一间 API 厂商能完美解决同一个 Prompt 产生相同角色的问题。
**架构师对策 (The "Reference Anchor" Pipeline)**：
我们在 `ComfyUI` 管线中强制引入 **IP-Adapter FaceID / Plus 模型**。
*   规则：第 1 页的生成物，自动经过 `YOLO` 或 `SAM` 裁切出主体，作为第 2、3 页的 `Image Prompt` 锚点。
*   妥协：我们在 Prompt 中强制加入 `3D claymation style (3D黏土风格)`。因为相较于写实风格，黏土风格的几何特征较少，AI 保持风格和特征一致的成功率会大幅提升 300%。

### 🚨 难点 3：Canvas 防误触 (Palm Rejection) 在 Web/跨平台上的先天缺陷
**问题**：iPad 的原生防误触依赖 Apple Pencil 的底层 API，而 React Native / Web 触控事件很容易把手掌压在萤幕上辨识为一团乱涂。
**架构师对策 (Heuristic Touch Filtering)**：
在前端拦截触控事件，写死一个启发式规则：
*   **判定 1**：如果触控点的接触面积（`touch.radiusX * touch.radiusY`）大于特定阈值（代表是手掌不是手指），直接 `event.preventDefault()`。
*   **判定 2**：如果同时出现 3 个以上的触控点，判定为手掌压著，暂停 Canvas 绘制输入。

## 四、 资源配置与 AI Coding 协同建议

为了将 8 周的时程压缩并确保高品质，团队不需要庞大，但需要「超级个体」配合 AI：

*   **全端工程师 A (偏前端 & 互动)**：1 名。专注于 React Native Skia、动画细节、录音模块。使用 `Cursor` 配合 `v0.dev` 快速生成 UI 骨架。
*   **AI 演算法与后端工程师 B (偏 Python & ComfyUI)**：1 名（通常是我自己）。专注于 FastAPI、Celery 伫列、ComfyUI 节点搭建、Prompt 系统调优。
*   **兼职 UI/UX 设计师**：提供高保真的 Figma 稿、色彩规范与 Loading 骨架图。

这份蓝图清楚界定了「理想（AI生成）」与「现实（逾时、破图）」的边界。我们的任务，就是用最严密的工程代码，把那些不完美的 AI 幻觉死死地挡在后台，呈现给孩子一个 100% 完美的魔法世界。**Let''s build it.**
